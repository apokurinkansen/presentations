# 事例3: PR TIMES — 発表者メモ

## 基本情報

- 発生日: 2025年4月24日（偵察は4月8〜9日）
- 発覚日: 2025年4月25日
- 公表日: 2025年5月7日
- 攻撃種別: 不正アクセス（バックドア設置によるデータ窃取）

## 攻撃の詳細

### 侵入経路

PR TIMES の管理者画面へのアクセスには以下の3段階の認証が必要:

1. **IP アドレス認証**
2. **BASIC 認証**
3. **ログインパスワード認証**

しかし、以下の2つの問題が重なり突破された:

- コロナ禍のリモート移行時にアクセス許可 IP を増やしたが、**追加経緯が不明な IP アドレス**が残存していた
- 認証に**普段使われていない共有アカウント**が使われた

### 時系列

| 日付 | 出来事 |
|------|--------|
| 4月8〜9日 | 偵察と推察される不正アクセス |
| 4月24〜25日 | 管理者画面への不正アクセス、バックドア設置 |
| 4月25日 | 不審ファイルを検知、調査開始 |
| 4月27〜28日 | 攻撃者による残存プロセスの実行を確認 |
| 4月28日 | 残存プロセスを特定・停止 |
| 4月30日 | バックドアプロセスを停止 |
| 5月7日 | 警察庁にサイバー攻撃被害を申告、公式発表 |

### 攻撃者の特徴

- Telegram 経由で通信
- 海外 IP からのアクセスも確認
- アクセス権限が別の攻撃者へ移行された可能性あり

### 悪用被害

- 顧客やシステムの実被害は確認されていない

## 再発防止策

1. 管理画面アクセスを社内・VPN 接続のみに制限
2. バックドア配置箇所で不正ファイル実行禁止設定を実装
3. WAF 設定見直し（予定）
4. 新管理者画面への移行（2025年中予定）— 共有アカウント廃止

## 研修で強調すべきポイント

- **「とりあえず」の設定変更が穴になる**: コロナ禍で一時的に IP 制限を緩和した設定がそのまま残存
- **共有アカウントの危険性**: 「誰が使ったか分からない」アカウントが攻撃者に悪用された
- **多層認証でも突破される**: 3段階の認証があっても、各層に弱点があれば意味がない
- **サービスは無事でもデータは漏れる**: サービス自体は正常稼働でも、90万件の情報が漏洩した
- **発表前のプレスリリースも漏洩**: 個人情報だけでなく、企業の未公開情報も漏洩リスクがある
