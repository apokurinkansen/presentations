# Slide Style Guide

## カラーパレット

| トークン | 用途 | Hex | プレビュー |
|--------|------|---------|---------|
| Background | スライド背景 | `#fffffe` | ほぼ白 |
| Text | 本文・h1 | `#0d0d0d` | ほぼ黒 |
| Text-sub | h2・h3 | `#2a2a2a` | ダークグレー |
| Surface | カード・パネル背景 | `#eff0f3` | ライトグレー |
| Stroke | ボーダー・区切り線 | `#0d0d0d` | ≒ Text |
| On-dark | 暗い背景上のテキスト | `#fffffe` | 白 |
| Highlight | アクセント・テーブルヘッダー | `#ff8e3c` | オレンジ |
| Secondary | 装飾背景 | `#eff0f3` | ≒ Surface |
| Tertiary | リンク・強調ボーダー | `#d9376e` | ディープピンク |
| Link | リンク | `#d9376e` | ≒ Tertiary |
| Blockquote-bg | 引用・コード背景 | `#eff0f3` | ≒ Surface |
| Progress Bar | ページ進捗バー | `#3c8eff` | 青（オレンジの補色） |

### 使い分けルール

- **Highlight (`#ff8e3c`)** はテーブルヘッダーや背景アクセントに使う。テキスト色には非推奨
- **Progress Bar (`#3c8eff`)** はインラインコード文字色・プログレスバー・`.panel-highlight` に使用
- 1スライドのアクセントカラーは **1〜2色** まで
- 装飾的な絵文字は使わない

## レイアウトパターン

### 2カラム

```markdown
:::c
左カラムの内容

右カラムの内容
:::
```

`:::col` も同じ動作。

### 3カラム

```markdown
:::c3
1列目

2列目

3列目
:::
```

### 左右比率指定（7:3）

```markdown
:::c-7-3
メインコンテンツ（広い）

サイドバー（狭い）
:::
```

`:::c-3-7` で逆比率。

### 2×2 グリッド

```markdown
:::grid-2x2
セル1

セル2

セル3

セル4
:::
```

### 汎用グリッド

```markdown
:::grid
左上

右上

左下

右下
:::
```

### ネスト

コロンの数を増やしてネストする:

```markdown
::::c
:::_ {.panel}
左のパネル
:::

:::_ {.panel-accent}
右のアクセントパネル
:::
::::
```

## コンポーネント

### パネル

```markdown
:::_ {.panel}
基本パネル（グレー背景）
:::

:::_ {.panel-accent}
アクセントパネル（左ボーダー: ダーク）
:::

:::_ {.panel-highlight}
ハイライトパネル（青背景・青ボーダー）
:::

:::_ {.panel-warn}
警告パネル（黄色系）
:::

:::_ {.panel-success}
成功パネル（緑系）
:::

:::_ {.panel-danger}
危険パネル（赤系）
:::
```

### アラート

```markdown
:::_ {.note}
補足情報
:::

:::_ {.tip}
ヒント
:::

:::_ {.important}
重要
:::

:::_ {.warning}
注意
:::

:::_ {.caution}
警告
:::
```

### インラインスタイル

```markdown
<!-- markdown-it-attrs でクラス付与 -->
大きいテキスト{.text-xl}

<!-- markdown-it-mark + attrs でインライン装飾 -->
この中の==ここだけ=={.bg-highlight}ハイライト

<!-- 色の変更 -->
==ディープピンクのテキスト=={.tertiary}
```

### コードブロック（ファイル名付き）

````markdown
```tsx {name=component.tsx}
const App = () => <div>Hello</div>;
```
````

## テキストルール

### 文体

- タイトルにコロン（:）を使わない
- 感嘆符（！）を避ける。理知的で装飾的でない文体を維持する
- 一文は簡潔に。体言止めを活用する

### 情報量の目安

- **箇条書き**: 1スライド 3〜5項目
- **テーブル**: 3〜4行 × 3列 程度
- **コードブロック**: 10行以内
- 収まらない場合はスライドを分割するか、カラムレイアウトを使う

## スライドの種類

| クラス | 用途 | 背景 |
|--------|------|------|
| `lead` | タイトルスライド | 淡いオレンジのグラデーション |
| `section` | セクション区切り | 淡いオレンジのグラデーション（逆方向） |
| （なし） | 通常コンテンツ | `#fffffe` |

```markdown
<!-- タイトルスライド -->
<!-- _class: lead -->
<!-- _paginate: false -->

# プレゼンタイトル

サブタイトル

---

<!-- セクション区切り -->
<!-- _class: section -->
<!-- _paginate: false -->

# セクション名
```

## プログレスバー

右端に青のプログレスバーが自動表示される（`paginate: true` 時）。Lead / Section スライドでは非表示。

## アンチパターン

- **Highlight をテキスト色に使う** → コントラスト不足で読めない。背景やボーダー専用
- **1スライドに3色以上のアクセント** → 散漫になる。1〜2色に絞る
- **情報を詰め込みすぎる** → 見切れる。分割するかレイアウトを変える
- **毎回新しいデザインを作る** → パターンから選ぶ
- **原色（.red, .blue 等）を多用する** → テーマカラーを使う

## チェックリスト

スライド作成後に確認する:

- [ ] すべてのスライドが見切れていないか（PDF で確認）
- [ ] カラーパレット外の色を使っていないか
- [ ] 1スライドの情報量は適切か（箇条書き5項目以内）
- [ ] Lead / Section スライドに正しいクラスが設定されているか
- [ ] 画像が鮮明か（ぼやけていないか）
- [ ] リンクに出典が記載されているか
